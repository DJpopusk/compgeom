# compgeom

Репозиторий с задачами по вычислительной геометрии. Для каждой задачи:

- **algorithms/** — чистый C++ (без Qt) с реализацией алгоритмов, оформленный как статические библиотеки.
- **qt/** — визуализаторы на Qt, которые просто используют функции из соответствующих библиотек.

Сейчас добавлены только первые две задачи из каталога пользователя:

| Задача | Алгоритмы | Qt-визуализация | Описание |
| ------ | ----------| --------------- | -------- |
| 1 | `algorithms/task1` | `qt/task1` | Отношение точки к направленному отрезку (лево/право/на отрезке с допуском ε). |
| 2 | `algorithms/task2` | `qt/task2` | Пересечение двух отрезков с учётом допуска ε. |

## Сборка

### Требования
- CMake 3.16+
- Компилятор с поддержкой C++17
- Для Qt-проектов: Qt 5 или Qt 6 (модуль Widgets)

### Команды
```bash
mkdir build && cd build
cmake ..            # найдёт Qt, соберёт библиотеки и визуализаторы
cmake --build .
```

Если нужно собрать только алгоритмы (без Qt), используйте опцию:
```bash
cmake -S . -B build -DCOMPGEOM_BUILD_QT=OFF
cmake --build build
```

## Запуск

После сборки исполняемые файлы появятся в каталоге сборки, например `build/qt/task1/task1_visual` и `build/qt/task2/task2_visual`.

### Task 1
- ЛКМ: ставит точки A и B (по кругу: A → B → A).
- ПКМ: ставит точку P.
- `+` / `-`: масштабирование, `0`: сброс.
- `[` / `]` / `E`: изменение допуска ε.

### Task 2
- ЛКМ: ставит вершины A,B,C,D (отрезки AB и CD).
- Флажок «Онлайн» позволяет перетаскивать вершины и видеть пересечение сразу.
- Кнопка «Вычислить» делает одиночный расчёт при выключенном онлайн-режиме.
- Поддерживаются `+`, `-`, `0` для масштабирования.

## Расширение
Чтобы добавить следующую задачу, поместите алгоритмы в `algorithms/taskN`, визуализацию в `qt/taskN`, подключите библиотеку в `qt/taskN/CMakeLists.txt` и зарегистрируйте подпроект в соответствующем `CMakeLists.txt`.
